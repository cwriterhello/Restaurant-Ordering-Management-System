# 餐饮管理系统功能设计与实现

## 4.1 用户点餐管理模块

### 4.1.1 功能描述

客户通过扫描桌号二维码或访问点餐页面进入点餐系统。在点餐首页，客户可以看到餐厅的所有菜品分类，包括热菜、凉菜、主食、饮品等。点击某个分类后，页面展示该分类下的所有菜品列表，每个菜品卡片显示菜品图片、名称、价格、库存状态和"加入购物车"按钮。

客户点击菜品卡片的"加入购物车"按钮，可以选择菜品数量，系统将菜品添加到购物车中。客户可以随时查看购物车，修改菜品数量或删除菜品。购物车底部显示菜品总数和合计金额。客户确认订单后，填写备注信息（如口味偏好、特殊要求等），点击"提交订单"按钮，系统在数据库中创建订单记录，订单状态初始化为"待处理"，并跳转到订单成功页面显示订单号和预计等待时间。

如果客户是会员，可以在提交订单前输入手机号，系统自动查询会员信息并应用对应的折扣率。点餐页面支持菜品搜索功能，客户可以输入菜品名称快速查找。对于库存不足的菜品，系统显示"已售罄"标识并禁用购买按钮。用户点餐页面如图4-1所示。

### 4.1.2 功能实现

①运用Vue 3的Composition API构建点餐页面组件，使用ref定义响应式数据categories存储菜品分类列表，dishes存储菜品列表，cartItems存储购物车商品。使用computed属性cartTotal计算购物车总金额，cartCount计算购物车商品总数。

②定义getCategoriesApi方法调用/api/categories接口获取菜品分类，定义getDishesByCategoryApi方法根据分类ID获取菜品列表。在onMounted生命周期钩子中初始化加载分类数据和默认分类的菜品数据。

③运用Pinia状态管理库创建cart store，定义addItem方法添加商品到购物车，removeItem方法从购物车移除商品，updateQuantity方法更新商品数量，clearCart方法清空购物车。购物车数据持久化存储在localStorage中，防止页面刷新丢失。

④定义createOrderApi方法调用POST /api/orders接口创建订单，请求体包含tableNumber、memberPhone、remark和items数组。提交成功后跳转到订单成功页面，显示订单号和二维码供客户查看订单状态。使用Element Plus的el-badge组件在购物车图标上显示商品数量角标。

## 4.2 后厨管理模块

### 4.2.1 功能描述

后厨人员或管理员在后厨管理页面可以看到三个标签页，分别为"待处理"、"制作中"和"已完成"。系统通过WebSocket实时推送新订单，当客户提交订单后，订单自动出现在"待处理"标签页中，显示订单号、桌号、下单时间、菜品清单和备注信息。

后厨人员点击"开始制作"按钮，订单状态从"待处理"变更为"制作中"，订单卡片移动至"制作中"标签页；点击"完成制作"按钮，订单状态从"制作中"变更为"已完成"，订单卡片移动至"已完成"标签页。每个订单卡片以网格布局展示，包含订单基本信息、菜品列表和操作按钮。系统会在订单状态变更时通过WebSocket通知其他客户端同步更新。后厨管理页面如图4-2所示。

### 4.2.2 功能实现

①运用Vue 3的ref和computed创建响应式数据，orders数组存储所有订单数据，pendingOrders、preparingOrders和readyOrders三个计算属性分别过滤出不同状态的订单。

②定义loadOrders方法调用getAllOrdersApi接口获取全量订单数据，定义handleUpdateStatus方法调用updateOrderStatusApi接口更新订单状态，更新成功后重新加载订单列表。

③运用SockJS和STOMP协议建立WebSocket连接，在onMounted生命周期钩子中调用connectWebSocket方法订阅/topic/kitchen/orders主题，接收后端推送的订单更新消息，实时更新本地订单数据并显示提示消息。

④使用Element Plus的el-tabs组件实现标签页切换，通过@tab-change事件监听标签页切换，触发订单数据刷新。使用自定义OrderList组件渲染订单列表，通过props传递订单数据，通过emit向父组件发送状态更新事件。

## 4.3 收银管理模块

### 4.3.1 功能描述

收银员或管理员在收银管理页面可以看到两个标签页，分别为"待结账订单"和"会员管理"。在待结账订单标签页中，系统展示所有状态为"已完成"且未支付的订单列表，每个订单卡片显示订单号、桌号、下单时间、菜品清单、订单总额、折扣金额和实付金额等信息。

收银员点击订单卡片下方的"现金支付"、"支付宝"或"微信支付"按钮，系统弹出确认对话框，确认后调用支付接口完成订单结算，订单状态变更为"已支付"，并从待结账列表中移除。在会员管理标签页中，收银员可以查看会员列表，支持按手机号搜索会员，可以为顾客注册新会员，设置会员等级和折扣率。收银管理页面如图4-3所示。

### 4.3.2 功能实现

①运用Vue 3的ref和computed创建响应式数据，orders数组存储待结账订单，通过computed属性readyOrders过滤出状态为READY的订单。定义activeTab变量控制标签页切换。

②定义loadOrders方法调用getReadyOrdersApi接口获取待结账订单列表，定义handlePay方法处理支付逻辑，接收orderId和paymentMethod参数，构造PaymentDTO对象调用payOrderApi接口完成支付。

③运用SockJS和STOMP协议建立WebSocket连接，订阅订单状态更新主题，当订单状态变更为READY时自动添加到待结账列表，当订单支付完成后从列表中移除。

④使用Element Plus的el-tabs组件实现标签页切换，集成OrderList组件展示订单列表，通过@pay事件监听支付操作。在会员管理标签页中嵌入MemberManagement组件，复用会员管理功能。

## 4.4 业务管理模块

业务管理模块包括菜品管理、套餐管理、会员管理和桌号管理四个子模块，涵盖餐厅日常运营的核心业务功能。

### 4.4.1 菜品管理

#### 4.4.1.1 功能描述

管理员在菜品管理页面可以查看餐厅所有菜品的列表信息，包括菜品ID、菜品名称、所属分类、单价、库存、单位、上下架状态、是否推荐、排序号和创建时间等字段。页面顶部提供搜索功能，管理员可以输入菜品名称进行模糊查询。

管理员点击"新增菜品"按钮，弹出表单对话框，填写菜品名称、选择分类、输入价格、库存、单位等信息后提交，系统在数据库中创建新的菜品记录。管理员点击菜品行的"编辑"按钮，可以修改菜品信息；点击"上架/下架"开关，可以切换菜品的可见状态；点击"删除"按钮，经二次确认后永久删除菜品记录。菜品列表采用分页展示，默认每页显示10条记录。菜品管理页面如图4-3所示。

#### 4.4.1.2 功能实现

①运用Spring Boot框架创建DishController控制器类，使用@RestController注解标识为RESTful API控制器，使用@RequestMapping("/api/dishes")定义基础路径。

②定义getAvailableDishes方法处理GET请求，调用dishService.getAvailableDishes()获取上架状态的菜品列表；定义getAllDishes方法获取所有菜品（含下架）；定义getDishById方法根据ID查询单个菜品；定义createDish方法处理POST请求创建新菜品；定义updateDish方法处理PUT请求更新菜品信息；定义updateDishStatus方法处理菜品上下架状态切换。

③运用MyBatis-Plus的IService接口和ServiceImpl实现类封装数据库操作，在DishService中定义业务逻辑方法，使用LambdaQueryWrapper构建查询条件，实现按分类筛选、按名称模糊查询等功能。

④前端使用Element Plus的el-table组件展示菜品列表，通过:data属性绑定菜品数据数组，使用el-table-column定义列，通过slot插槽自定义状态列和操作列的渲染逻辑。使用el-dialog组件实现新增和编辑表单，通过el-form-item定义表单项，使用el-select、el-input-number等组件构建表单控件。

### 4.4.2 套餐管理

#### 4.4.2.1 功能描述

管理员在套餐管理页面可以查看套餐列表，包括套餐ID、套餐名称、描述、图片、价格、原价、上下架状态和创建时间等字段。管理员可以点击"新增套餐"按钮创建新套餐。

在新增套餐对话框中，管理员首先填写套餐基本信息，包括套餐名称、描述、价格等，然后在菜品选择区域可以看到所有可用菜品的列表，每个菜品前有复选框和数量输入框。管理员勾选需要的菜品并设置数量，系统自动计算套餐总价。点击"确定"按钮后，系统先创建套餐记录，再批量创建套餐与菜品的关联关系。管理员可以编辑已有套餐信息和菜品配置，可以切换套餐的上下架状态。套餐管理页面如图4-4所示。

#### 4.4.2.2 功能实现

①运用Spring Boot框架创建ComboController控制器类，定义getAllCombos方法获取套餐列表，定义getComboWithDishes方法根据套餐ID查询套餐及其关联菜品，定义createComboWithDishes方法处理套餐和菜品的批量创建。

②在createComboWithDishes方法上添加@Transactional注解确保事务一致性，先调用comboService.save()保存套餐基本信息获取生成的套餐ID，再遍历请求中的菜品配置列表，为每个菜品创建ComboDish关联记录，设置套餐ID、菜品ID和数量。

③前端使用el-checkbox-group组件实现菜品多选，使用el-input-number组件设置每个菜品的数量。定义selectedDishes数组存储选中的菜品ID，定义dishQuantities对象存储每个菜品的数量。在提交时构造ComboAndDishesRequest对象，包含套餐基本信息和菜品配置列表。

④使用Element Plus的el-card组件展示套餐卡片，通过v-for指令遍历套餐列表，在卡片中显示套餐图片、名称、价格和包含的菜品清单。使用el-tag组件展示菜品数量和价格信息。

### 4.4.3 会员管理

#### 4.4.3.1 功能描述

管理员在会员管理页面可以查看会员列表，包括会员ID、手机号、姓名、会员等级、折扣率、积分、累计消费金额和注册时间等字段。页面顶部提供搜索面板，管理员可以输入手机号进行精确查询。

管理员点击"新增会员"按钮，弹出表单对话框，填写手机号、姓名、选择会员等级、设置折扣率后提交，系统在数据库中创建新的会员记录。会员等级分为普通会员、银卡会员、金卡会员和白金会员四个等级，折扣率范围为0.1至1.0。管理员点击会员行的"编辑"按钮，可以修改会员信息，但手机号不可修改。会员管理页面如图4-5所示。

#### 4.4.3.2 功能实现

①运用Spring Boot框架创建MemberController控制器类，定义getMembers方法获取会员列表，定义createMember方法处理POST请求创建新会员，定义updateMember方法处理PUT请求更新会员信息，定义getMemberByPhone方法根据手机号查询会员。

②在createMember方法中，使用@RequestBody注解接收JSON格式的Member对象，验证手机号必填，设置默认等级为NORMAL、默认折扣为1.00，调用memberService.createMember()创建会员记录。如果传入了自定义等级和折扣，则调用updateMember方法更新这些信息。

③前端使用el-button组件添加"新增会员"按钮，绑定handleAdd方法打开对话框。定义isEdit响应式变量区分新增和编辑模式，在新增模式下清空表单并设置默认值，在编辑模式下填充现有会员数据。

④使用el-form组件构建会员表单，定义memberRules验证规则，对手机号使用正则表达式/^1[3-9]\d{9}$/验证格式，对姓名、等级和折扣设置必填验证。在handleSave方法中调用formRef.validate()执行表单验证，验证通过后根据isEdit状态调用createMemberApi或updateMemberApi接口。

### 4.4.4 桌号管理

#### 4.4.4.1 功能描述

管理员在桌号管理页面可以查看所有桌号的列表，包括桌号ID、桌号编号、容纳人数、当前状态和二维码等信息。系统为每个桌号自动生成唯一的二维码，客户扫描后可以进入点餐页面。

管理员可以点击"生成二维码"按钮为指定桌号生成二维码图片，支持批量生成和下载。二维码内容为点餐页面的URL，包含桌号参数，例如http://localhost:5173/customer/table/A01。管理员可以查看桌号的当前状态（空闲、占用、预订），可以修改桌号的容纳人数。桌号管理页面如图4-6所示。

#### 4.4.4.2 功能实现

①运用Spring Boot框架创建TableController控制器类，定义getAllTables方法获取所有桌号，定义getTableByNumber方法根据桌号查询，定义validateTable方法验证桌号有效性。

②使用qrcode.vue库在前端生成二维码，定义generateQRCode方法，传入桌号对应的点餐URL，设置二维码尺寸为200x200像素，纠错级别为M。使用canvas元素渲染二维码，支持导出为PNG图片。

③定义Table实体类，包含id、tableNumber、qrCode、capacity、status等字段。使用MyBatis-Plus的@TableId注解配置主键自增，使用@TableField注解映射数据库字段。

④前端使用el-image组件展示二维码图片，使用el-popover组件实现鼠标悬停放大查看。定义downloadQRCode方法，使用a标签的download属性实现二维码图片下载。

## 4.5 系统管理模块

系统管理模块包括用户管理和数据统计两个子模块，提供系统层面的用户权限管理和数据分析功能。

### 4.5.1 用户管理

#### 4.5.1.1 功能描述

超级管理员在用户管理页面可以查看系统用户列表，包括用户ID、用户名、真实姓名、角色、创建时间和最后登录时间等字段。管理员可以点击"新增用户"按钮创建新用户账号。

在新增用户对话框中，管理员填写用户名、密码、确认密码、真实姓名，选择角色（管理员、收银员或后厨），点击"确定"后系统在数据库中创建用户记录，密码使用BCrypt算法加密存储。管理员可以编辑用户信息、重置密码、删除用户账号，但不能删除自己的账号。普通管理员只能创建收银员和后厨角色，不能创建管理员角色。用户管理页面如图4-7所示。

#### 4.5.1.2 功能实现

①运用Spring Boot框架创建UserController控制器类，定义getAllUsers方法获取用户列表，定义createUser方法创建新用户，定义updateUser方法更新用户信息，定义resetPassword方法重置用户密码，定义deleteUser方法删除用户。

②在createUser方法中，首先验证角色权限，如果当前用户是ADMIN且尝试创建ADMIN角色则拒绝操作。使用BCryptPasswordEncoder对用户密码进行加密，调用userService.saveUser()保存用户记录。

③前端使用el-table展示用户列表，在操作列根据当前用户ID判断是否显示删除按钮，防止用户删除自己。在新增表单中使用el-input组件的type="password"属性隐藏密码输入，添加密码强度提示。

④定义角色映射对象roleMap，将ADMIN、CASHIER、KITCHEN转换为中文显示。在表格中使用el-tag组件展示角色，根据不同角色设置不同的tag类型（danger、primary、success）。

### 4.5.2 数据统计

#### 4.5.2.1 功能描述

管理员在数据统计页面可以查看营业数据的可视化展示，包括今日订单数、今日营业额、本月订单数、本月营业额等关键指标卡片。页面中部展示近7天的订单趋势折线图和支付方式分布饼图。

页面底部展示库存预警列表，显示当前库存低于预警阈值的菜品，包括菜品名称、当前库存、预警阈值和预警状态。管理员可以点击"处理"按钮标记预警为已处理，系统记录处理时间。数据统计页面如图4-8所示。

#### 4.5.2.2 功能实现

①运用Spring Boot框架创建StatisticsController控制器类，定义getDailyStatistics方法获取每日统计数据，定义getStockAlerts方法获取库存预警列表。

②定义DailyStatistics实体类，包含statDate、totalOrders、totalAmount、cashAmount、alipayAmount、wechatAmount、memberAmount等字段。系统每天凌晨自动统计前一天的营业数据并插入统计表。

③定义StockAlert实体类，包含dishId、dishName、currentStock、alertThreshold、status等字段。在DishService的updateStock方法中，每次扣减库存后检查是否低于阈值，如果低于则创建或更新预警记录。

④前端使用ECharts图表库绘制折线图和饼图，定义initChart方法初始化图表实例，设置option配置项，包括xAxis、yAxis、series等。使用el-statistic组件展示关键指标卡片，设置title、value和suffix属性。

### 4.6 关键代码实现

各管理模块的核心控制器代码结构如下：

#### （1）菜品管理控制器（DishController）

```
@RestController
@RequestMapping("/api/dishes")
public class DishController {
    /**
     * 接口
     * GET /api/dishes:查询可用菜品
     * GET /api/dishes/all:查询所有菜品
     * GET /api/dishes/{id}:查询单个菜品
     * GET /api/dishes/category/{categoryId}:按分类查询
     * GET /api/dishes/recommend:查询推荐菜品
     * POST /api/dishes:新增菜品
     * PUT /api/dishes/{id}:修改菜品
     * PUT /api/dishes/{id}/status:更新上下架状态
     * DELETE /api/dishes/{id}:删除菜品
     */
    @Autowired
    private DishService dishService;
    
    @GetMapping
    public ApiResponse<List<DishVO>> getAvailableDishes() {
        List<DishVO> dishes = dishService.getAvailableDishes();
        return ApiResponse.success(dishes);
    }
    
    @PostMapping
    public ApiResponse<DishVO> createDish(@Valid @RequestBody DishDTO dishDTO) {
        DishVO savedDish = dishService.saveDish(dishDTO);
        return ApiResponse.success("菜品创建成功", savedDish);
    }
    
    @PutMapping("/{id}")
    public ApiResponse<DishVO> updateDish(@PathVariable Long id, @Valid @RequestBody DishDTO dishDTO) {
        dishDTO.setId(id);
        DishVO updatedDish = dishService.saveDish(dishDTO);
        return ApiResponse.success("菜品更新成功", updatedDish);
    }
    
    @PutMapping("/{id}/status")
```

#### （5）用户服务层核心业务逻辑（UserService）

```java
@Service
public class UserService {
    
    @Autowired
    private UserMapper userMapper;
    
    @Autowired
    private PasswordEncoder passwordEncoder;
    
    /**
     * 获取所有用户列表
     * 转换为VO对象，隐藏敏感信息（如密码）
     */
    public List<UserVO> getAllUsers() {
        return userMapper.selectList(null).stream()
                .map(this::convertToVO)
                .collect(Collectors.toList());
    }
    
    /**
     * 保存新用户
     * 使用BCrypt算法加密密码，确保安全性
     */
    public User saveUser(User user) {
        // 加密密码
        if (user.getPassword() != null && !user.getPassword().isEmpty()) {
            user.setPassword(passwordEncoder.encode(user.getPassword()));
        }
        userMapper.insert(user);
        return user;
    }
    
    /**
     * 更新用户信息
     * 如果未提供新密码，则保持原密码不变
     */
    public User updateUser(User user) {
        // 查询现有用户
        User existingUser = userMapper.selectById(user.getId());
        
        if (user.getPassword() == null || user.getPassword().isEmpty()) {
            // 保持原密码
            user.setPassword(existingUser.getPassword());
        } else {
            // 加密新密码
            user.setPassword(passwordEncoder.encode(user.getPassword()));
        }
        
        userMapper.updateById(user);
        return user;
    }
    
    /**
     * 重置用户密码
     * 管理员可为忘记密码的用户重置密码
     */
    public void resetPassword(Long id, String newPassword) {
        User user = userMapper.selectById(id);
        if (user != null) {
            user.setPassword(passwordEncoder.encode(newPassword));
            userMapper.updateById(user);
        }
    }
    
    /**
     * Entity转VO，隐藏敏感字段
     */
    private UserVO convertToVO(User user) {
        if (user == null) {
            return null;
        }
        UserVO vo = new UserVO();
        vo.setId(user.getId());
        vo.setUsername(user.getUsername());
        vo.setRealName(user.getRealName());
        vo.setRole(user.getRole());
        vo.setPhone(user.getPhone());
        vo.setStatus(user.getStatus());
        vo.setCreateTime(user.getCreateTime());
        vo.setUpdateTime(user.getUpdateTime());
        // 注意：不返回password字段，保障安全性
        return vo;
    }
}
```

以上各控制器均采用RESTful API设计规范，使用@RestController注解声明REST接口，通过@RequestMapping定义基础路径，利用@PathVariable和@RequestParam处理路径参数和查询参数，通过@RequestBody接收JSON格式请求体。所有接口统一返回ApiResponse对象，包含code、message和data三个字段，确保前后端数据交互的一致性。
