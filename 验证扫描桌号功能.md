# 扫描桌号验证功能实现总结

## 功能描述
实现了扫描桌号时判断桌号是否存在的功能，当用户扫描桌号二维码或手动输入桌号时，系统会验证该桌号是否存在，如果不存在则给出相应提示。

## 已完成的实现

### 1. 后端实现
- 在 `TableController.java` 中添加了验证端点：`GET /api/tables/{tableNumber}/validate`
- 端点返回 `ApiResponse<Boolean>`，表示桌号是否存在

### 2. 前端实现
- 在 `frontend/src/api/table.ts` 中添加了 `validateTableNumberApi` 函数
- 在 `frontend/src/views/customer/Index.vue` 中修改了 `goToTable` 函数，跳转前验证桌号
- 在 `frontend/src/views/customer/TableOrder.vue` 中添加了页面加载时的桌号验证逻辑

## 验证逻辑

### 手动输入桌号验证 (CustomerIndex.vue)
1. 用户在首页输入桌号
2. 点击"确认"按钮时，调用 `validateTableNumberApi` 验证桌号
3. 如果桌号存在，跳转到对应桌号的点餐页面
4. 如果桌号不存在，显示错误提示

### 扫描桌号验证 (TableOrder.vue)
1. 用户通过扫描二维码直接访问桌号页面
2. 页面加载时自动调用 `validateTableNumberApi` 验证桌号
3. 如果桌号存在，正常显示点餐页面
4. 如果桌号不存在，显示错误提示并跳转回首页

## API 端点
- 后端: `GET /api/tables/{tableNumber}/validate`
- 前端: `validateTableNumberApi(tableNumber: string): Promise<ApiResponse<boolean>>`

## 用户体验改进
- 防止用户访问不存在的桌号
- 提供友好的错误提示
- 自动跳转到首页重新输入桌号
